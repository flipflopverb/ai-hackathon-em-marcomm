# Milestones

## Purpose
Track delivery phases and completion criteria.

## Inputs
- `PLAN/PLAN.md`
- `../team/HANDOFF.md`

## Outputs
- Shared milestone outline and done definitions.

## Milestones

### Milestone 1: Foundation

Done when:
- stack and plan are stable
- gameplay and handoff contracts are stable
- repo structure supports `src/`, `assets/`, and specs
- GitHub Pages deployment path is defined

### Milestone 2: First Playable

Done when:
- game boots locally
- player can move through one map
- collisions and trigger zones work
- at least 3 landmark interactions complete end-to-end
- `Swoop` growth visibly updates

### Milestone 3: Full v1 Loop

Done when:
- 5-8 landmark interactions are present
- one full run resolves one `academic_interest`
- result screen shows `Swoop` state and collectibles
- Slate payload builder works with the required fields

### Milestone 4: Demo and Launch Readiness

Done when:
- GitHub Pages build is live and working
- core path works on desktop and mobile
- major edge cases have been checked
- demo path is stable and repeatable

## Constraints
- Keep milestones realistic for hackathon timelines.
