# Project Brief

## Purpose
Describe the problem, audience, and value of the v1 campus exploration game.

## Inputs
- `PLAN/PLAN.md`
- `CONTEXT.md`
- `SCOPE.md`

## Outputs
- Clear project narrative for stakeholders, judges, and demo framing.

## Problem

Prospective students are used to institutional experiences that feel static, generic, or form-first. That makes it harder to create a memorable emotional connection to campus before a student ever visits.

## Audience

Primary audience:
- prospective students exploring whether the University of Utah feels like a fit

Secondary audience:
- stakeholders reviewing the concept as an enrollment-marketing experiment

## v1 Solution

Create a short browser-based campus exploration game where the player walks through a stylized map, discovers landmarks, grows `Swoop`, collects campus items, and ends with an `academic_interest` result plus an optional Slate handoff.

## Value Proposition

The experience should:
- make campus feel more alive and memorable
- turn student-interest discovery into play rather than a questionnaire
- create a clearer emotional payoff through `Swoop` growth
- produce one useful handoff signal for Slate without overcomplicating CRM integration

## Constraints
- Keep aligned to current v1 scope boundaries.
- Keep the first release to one playable vertical slice.
- Keep the final deployment compatible with GitHub Pages.

## Open Questions
- Which student motivations should the live demo emphasize most: place, academics, or student life?
