# Scope (Stakeholder View)

## Purpose
Summarize what is in and out of v1 in plain language.

## Inputs
- `PLAN/PLAN.md` (canonical)
- `PROJECT-BRIEF.md`

## Outputs
- Non-technical scope summary for stakeholders.

## In Scope

- One short campus exploration game
- One stylized campus map
- 5-8 landmark interactions
- A growing `Swoop` companion
- Collectible campus items
- One `academic_interest` result
- One optional Slate handoff after results
- Static deployment via GitHub Pages

## Out Of Scope

- Multiple maps
- Combat or RPG battle systems
- Deep branching storylines
- Large open-world simulation
- Multiple audience-specific game versions
- In-game collection of student contact information
- CRM logic beyond the final Slate handoff

## Constraints
- Must not conflict with `PLAN/PLAN.md`.
- Must stay realistic for a hackathon-scale build.

## Open Questions
- Which out-of-scope items are most likely follow-on work after the first playable slice?
